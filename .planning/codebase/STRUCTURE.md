## Directory Purposes

**extension/:**
- Purpose: Chrome Extension source code (all user-facing code, extension APIs)
- Contains: Content script, background worker, sidebar UI, settings page, WASM glue code
- Loads into Chrome via chrome://extensions → Load unpacked

**extension/sidebar/:**
- Purpose: Primary UI component for user interaction
- Contains: HTML structure, main JavaScript entry point, modular components
- Responsible for: Displaying analysis results, managing sessions, handling user settings

**extension/sidebar/modules/:**
- Purpose: Encapsulated JavaScript modules (ES6 classes)
- Contains: SessionManager (session lifecycle), StateManager (centralized state)
- Pattern: Each module is a class with focused responsibilities, exported as ES6 module

**extension/sidebar/utils/:**
- Purpose: Reusable helper functions and utilities
- Contains: DOM helpers (safe manipulation, XSS protection), validation, formatting
- Pattern: Pure functions or stateless utilities, no side effects (except DOM operations)

**extension/options/:**
- Purpose: Extension options page (user-configurable settings)
- Contains: HTML form, JavaScript logic, CSS styling
- Accessible via: chrome://extensions → Chat Signal Radar → Options button

**extension/libs/web-llm/:**
- Purpose: Optional in-browser LLM library for AI features
- Contains: WebLLM bundle (pre-built or user-downloaded)
- Status: Not committed to git (user downloads from WebLLM CDN if they enable AI)

**wasm-engine/:**
- Purpose: Rust-based analysis engine compiled to WebAssembly
- Contains: Message clustering, topic extraction, sentiment analysis logic
- Compiled to: `extension/wasm/wasm_engine.js` + `wasm_engine_bg.wasm`

**wasm-engine/src/:**
- Purpose: Source Rust code for WASM engine
- Contains: lib.rs (all analysis functions, data structures, word lists)
- Pattern: Exports functions with #[wasm_bindgen] macro for JS interop

**scripts/:**
- Purpose: Build automation and development tools
- Contains: Shell scripts for compilation and watching
- Usage: `./scripts/build.sh` (production), `./scripts/watch.sh` (development)

**.planning/codebase/:**
- Purpose: GSD mapping documents (generated by `/gsd:map-codebase`)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, etc.
- Not committed: Auto-generated from codebase exploration

## Naming Conventions

**Files:**

- **JavaScript:** camelCase (e.g., `sidebar.js`, `SessionManager.js`, `DOMHelpers.js`)
- **HTML:** kebab-case or descriptive (e.g., `sidebar.html`, `options.html`)
- **CSS:** kebab-case selectors (e.g., `.mood-section`, `.cluster-bucket`, `.hidden`)
- **Rust:** snake_case (e.g., `lib.rs`, `wasm_engine` module name)
- **Generated:** snake_case with _bg suffix for WASM (e.g., `wasm_engine_bg.wasm`)

**Directories:**

- **Camelcase for components:** `sidebar/`, `modules/`, `utils/`
- **Kebab-case for feature groups:** `web-llm/` (though typically camelCase in JS)
- **Lowercase for logical groups:** `icons/`, `libs/`, `options/`, `wasm/`, `scripts/`, `tests/`

**JavaScript Classes:**

- PascalCase for class names: `StateManager`, `SessionManager`
- Private methods prefixed with underscore: `_initializeWASM()`
- Constants in UPPER_SNAKE_CASE: `MAX_SESSION_QUESTIONS`, `SENTIMENT_UPDATE_INTERVAL`

**DOM Element IDs:**

- Kebab-case prefixed by feature: `mood-section`, `topics-cloud`, `ai-summary`, `llm-consent-modal`
- Semantic suffixes: `-text`, `-btn`, `-div`, `-list`, `-empty`

**WASM Functions:**

- snake_case exported with #[wasm_bindgen]: `cluster_messages`, `extract_topics`, `analyze_sentiment_signals`

## Where to Add New Code

**New UI Feature:**
1. Add HTML to `extension/sidebar/sidebar.html` (new section with appropriate ID)
2. Add styles to `extension/sidebar/sidebar.css` (use CSS variables for colors)
3. Add JavaScript to `extension/sidebar/sidebar.js` or new module in `extension/sidebar/modules/`
4. If needs state, add getters/setters to `extension/sidebar/modules/StateManager.js`
5. Add DOM helper if complex manipulation needed

**New Analysis Algorithm:**
1. Implement function in `wasm-engine/src/lib.rs`
2. Export with `#[wasm_bindgen]` macro
3. Add unit tests in same file (module at bottom: `#[cfg(test)]`)
4. Run `./scripts/build.sh` to compile to WASM
5. Call from `extension/sidebar/sidebar.js` when processing messages

**New Settings Parameter:**
1. Add input to `extension/options/options.html`
2. Add getter/setter to `extension/options/options.js`
3. Add to DEFAULT_SETTINGS constant (must match in `extension/sidebar/sidebar.js`)
4. Add to `extension/sidebar/modules/StateManager.js` if sidebar needs to use it
5. Load from chrome.storage.sync on startup

**New Platform Support:**
1. Add platform detection to `extension/content-script.js` (e.g., check for kick.com)
2. Add DOM selectors for new platform's chat (YouTubeChat, TwitchChat patterns)
3. Add extract function: `extractKickMessage(element)`
4. Add to isYouTube/isTwitch conditional logic
5. Update manifest.json host_permissions to include new domain
6. Test in browser: Load extension, navigate to new platform

**Utility Functions:**
- DOM utilities: Add to `extension/sidebar/utils/DOMHelpers.js`
- Validation logic: Add to `extension/sidebar/utils/ValidationHelpers.js`
- Formatting: Add to `extension/sidebar/utils/FormattingHelpers.js`

**Test Coverage:**
- Unit tests: `wasm-engine/src/lib.rs` (Rust tests in `#[cfg(test)]` module at bottom)
- Integration tests: `tests/` directory (JavaScript tests if any)
- Run tests: `cd wasm-engine && cargo test`

## Special Directories

**extension/wasm/:**
- Purpose: Generated WASM artifacts
- Generated: By `wasm-pack build` (called from scripts/build.sh)
- Committed: No (.gitignored)
- Rebuilt: Every `./scripts/build.sh` run
- Contents: wasm_engine.js (JS glue), wasm_engine_bg.wasm (binary)

**extension/libs/web-llm/:**
- Purpose: Optional third-party WebLLM library
- Generated: User downloads from CDN if enabling AI summaries
- Committed: No (.gitignored)
- Optional: Extension works without it (uses fallback summarizer)
- Setup: See extension/WEBLLM_SETUP.md for download instructions

**wasm-engine/target/:**
- Purpose: Rust build artifacts (debug, release, wasm32-unknown-unknown)
- Generated: By cargo build
- Committed: No (.gitignored)
- Cleaned: `cargo clean` removes entire directory

**.planning/codebase/:**
- Purpose: GSD-generated mapping documents
- Generated: By `/gsd:map-codebase` command
- Committed: Yes (adds to git after generation)
- Consumed by: `/gsd:plan-phase`, `/gsd:execute-phase`
