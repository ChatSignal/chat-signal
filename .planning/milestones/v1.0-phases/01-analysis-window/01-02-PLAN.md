  <action>
Three changes to sidebar.js:

**Change 1 — Remove hardcoded constant (line ~681):**
Remove this line entirely:
```javascript
const MAX_MESSAGES = 100; // Keep last 100 messages
```
There is no replacement constant. The window size comes from `settings.analysisWindowSize` at runtime.

**Change 2 — Update the message accumulator block (lines ~722-731):**
Locate the block inside the `chrome.runtime.onMessage` handler that currently reads:
```javascript
// Add new messages to accumulator
allMessages.push(...message.messages);

// Keep only recent messages
if (allMessages.length > MAX_MESSAGES) {
  allMessages = allMessages.slice(-MAX_MESSAGES);
}

// Process all accumulated messages
processMessages(allMessages);
```

Replace with:
```javascript
// Add new messages to accumulator
allMessages.push(...message.messages);

// Cap buffer at 2x window size to prevent unbounded growth
// (keeping 2x allows smooth window expansion without data loss)
const windowSize = settings.analysisWindowSize || 500;
if (allMessages.length > windowSize * 2) {
  allMessages = allMessages.slice(-(windowSize * 2));
}

// Slice to analysis window and process
const windowMessages = allMessages.slice(-windowSize);
processMessages(windowMessages);
```

Key invariant: `allMessages` is the full buffer (capped at 2x); `windowMessages` is what the WASM engine sees. The buffer is NOT cleared when the user shrinks the window — the next call to `processMessages` simply passes a shorter slice. When the user grows the window, any messages still in the 2x buffer are immediately available.

**Change 3 — Update processMessages() to display window stats:**
In the `processMessages()` function, after the line that sets `processedCount.textContent = totalMessageCount` (look for this existing line), add:
```javascript
// Update window stats indicator
const windowCurrentEl = document.getElementById('window-current');
const windowMaxEl = document.getElementById('window-max');
if (windowCurrentEl) windowCurrentEl.textContent = messages.length;
if (windowMaxEl) windowMaxEl.textContent = settings.analysisWindowSize || 500;
```

The `messages` parameter to `processMessages` is already `windowMessages` after Change 2, so `messages.length` correctly represents the current window fill level.

**Do NOT** reset `allMessages = []` anywhere in this task. The buffer persists across window size changes.
