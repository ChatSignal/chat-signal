<task type="auto">
  <name>Task 2: Create encoder-adapter.js with full encoding pipeline</name>
  <files>
    extension/sidebar/encoder-adapter.js
  </files>
  <action>
Create `extension/sidebar/encoder-adapter.js` as an ES6 module following the existing `llm-adapter.js` pattern. This module is the encoder singleton — created once, reused across all encode calls.

**Module structure:**

1. **Imports and env configuration (top of file, before any function):**
   ```javascript
   import { env, pipeline } from '../libs/transformers/transformers.js';
   env.backends.onnx.wasm.wasmPaths = chrome.runtime.getURL('libs/transformers/');
   env.allowRemoteModels = true;
   env.useBrowserCache = true;
   ```
   This MUST run before any pipeline() call (per Pitfall 2 in research).

2. **State variables:**
   - `encoderPipeline` — null until init completes
   - `encoderState` — `'idle'` | `'loading'` | `'ready'` | `'error'`
   - `backendUsed` — `'webgpu'` | `'wasm'` | `null`
   - `embeddingCache` — `new Map()` for hash→embedding storage
   - `MAX_CACHE_SIZE` — 2000 (per research recommendation, ~3MB max)
   - Batch queue variables: `encodingQueue`, `flushTimer`
   - Batch constants: `MIN_BATCH = 10`, `MAX_BATCH = 50`, `TIME_FLUSH_MS = 8000` (Claude's discretion: 8s timeout for slow chats)

3. **`initEncoder(onProgress)` — async, exported:**
   - Guard: if state is 'ready', return pipeline immediately; if 'loading', return existing promise (avoid double-init)
   - Set state to 'loading'
   - Detect WebGPU: check `navigator.gpu`, then `requestAdapter()`. If adapter exists, use `device: 'webgpu'`. Otherwise `device: 'wasm'`. Store in `backendUsed`.
   - Create pipeline: `await pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2', { device, dtype: 'q8', progress_callback })`.
   - **CRITICAL fallback (per Pitfall 3):** Wrap the WebGPU pipeline creation in try/catch. On failure, log `[Encoder] WebGPU init failed, falling back to WASM`, set `backendUsed = 'wasm'`, retry with `device: 'wasm'`.
   - progress_callback: Defensive null-checks on `event.status` and `event.progress` (per Pitfall 4). Forward to `onProgress` callback.
   - After pipeline created, run warm-up: `await encoderPipeline(['warm up'], { pooling: 'mean', normalize: true })` — primes GPU/WASM JIT.
   - Call `onProgress({ status: 'ready', progress: 100 })` after warm-up.
   - Set state to 'ready', return pipeline.

4. **`initEncoderWithRetry(onProgress, onError)` — async, exported:**
   - MAX_RETRIES = 3
   - Loop: try initEncoder(onProgress), catch → call `onError('Download failed, retrying... (N/3)')`, wait with exponential backoff (1s * attempt)
   - After 3 failures: call `onError('Semantic engine unavailable — using keyword analysis')`, set state to 'error', return null

5. **`simpleHash(str)` — internal:**
   - djb2-style hash returning string (`.toString(36)`)
   - Fast, no library needed for short chat messages

6. **`encodeMessages(messages)` — async, exported:**
   - Guard: if encoderPipeline is null, return null (encoder not ready)
   - Filter new messages: `messages.filter(m => !embeddingCache.has(simpleHash(m.text)))`
   - If new messages exist, batch-encode: `await encoderPipeline(texts, { pooling: 'mean', normalize: true })`, then `output.tolist()` to get `number[][]`
   - Store each new embedding in cache keyed by hash
   - Call `trimCache()` after storing
   - Return array of embeddings for ALL input messages (cached + new), indexed to match input order
   - Console.log cache stats: `[Encoder] Encoded ${newCount} new, ${cachedCount} cached`

7. **`trimCache()` — internal:**
   - If `embeddingCache.size > MAX_CACHE_SIZE`, delete oldest entries (Map preserves insertion order) until at `MAX_CACHE_SIZE`

8. **`scheduleEncode(messages, onBatchReady)` — exported:**
   - Push messages into `encodingQueue`
   - If queue length >= MAX_BATCH, call `flushQueue(onBatchReady)`
   - Otherwise reset timer: `clearTimeout(flushTimer)`, set new timeout for `TIME_FLUSH_MS` calling `flushQueue(onBatchReady)`

9. **`flushQueue(onBatchReady)` — internal:**
   - Clear timer if set
   - If queue empty, return
   - Splice up to MAX_BATCH from front of queue
   - Call `encodeMessages(batch)` then pass embeddings to `onBatchReady(batch, embeddings)`

10. **`getEncoderState()` — exported:** Returns `encoderState` string.

11. **`getBackendInfo()` — exported:** Returns `{ state: encoderState, backend: backendUsed }` — consumed by Settings page in Plan 02.

12. **`resetEncoder()` — exported:**
    - Dispose pipeline if exists (Transformers.js pipelines don't have explicit dispose, but set to null)
    - Clear embeddingCache
    - Clear encodingQueue and flushTimer
    - Reset state to 'idle'

**Logging:** All console.log prefixed with `[Encoder]`. No debug flag needed — locked decision says all encoder logs go to console.log only.
