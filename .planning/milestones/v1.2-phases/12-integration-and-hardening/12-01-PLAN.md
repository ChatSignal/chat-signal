**sidebar.js — Fix the WASM rendering gate (line ~544):**

The current gate at `processMessages()` returns early when `getEncoderState() === 'loading'`, which prevents WASM clusters from rendering during encoder download (up to 30s blank display). Per CONTEXT.md locked decision: "WASM works immediately" and per 10-02 decision: "WASM renders clusters synchronously for immediate display."

Replace the early-return gate with a skip-encode-only gate:

Current code (around line 544):
```javascript
if (!encoderReady && getEncoderState() === 'loading') {
  return;
}
```

Change to:
```javascript
const encoderLoading = !encoderReady && getEncoderState() === 'loading';
```

Then, later in the function where `scheduleEncode` is called (the async semantic routing callback), wrap it with:
```javascript
if (!encoderLoading) {
  scheduleEncode(windowMessages, (batch, embeddings, durationMs) => {
    // ... existing cosine routing callback ...
  });
}
```

This allows WASM keyword clusters, topics, mood, and stats to render normally during encoder loading. Only the `scheduleEncode` call is skipped (since the encoder isn't ready). Once the encoder finishes, subsequent `processMessages` calls will include `scheduleEncode`.

**llm-adapter.js — Add Qwen auto-retry after garbage fallback:**

Add module-level state:
```javascript
let _autoRetryScheduled = false;
const GARBAGE_RETRY_COOLDOWN_MS = 60_000;
```

In the `analyzeSentiment()` function, after the existing garbage fallback block (`if (_garbageCount >= MAX_GARBAGE_BEFORE_FALLBACK)`), add auto-retry scheduling. The auto-retry should ONLY schedule if the engine was a real Qwen engine (not already a fallback engine from missing bundle). Check `!engine._isFallback` BEFORE the fallback switch to determine this.

Implementation:
1. Before `_inFallback = true; engine = createFallbackEngine();` — capture `const wasRealEngine = engine && !engine._isFallback;`
2. After `engine = createFallbackEngine();` — add:
```javascript
if (wasRealEngine && !_autoRetryScheduled) {
  _autoRetryScheduled = true;
  setTimeout(async () => {
    _autoRetryScheduled = false;
    _inFallback = false;
    _garbageCount = 0;
    engine = null;
    isInitialized = false;
    isInitializing = false;
    try {
      await initializeLLM();
      if (engine && engine._isFallback) {
        _inFallback = true;
      }
    } catch (_) {
      _inFallback = true;
    }
  }, GARBAGE_RETRY_COOLDOWN_MS);
}
```

This ensures: (a) auto-retry only happens when a real Qwen engine produced garbage (not when bundle is missing), (b) only one retry is scheduled (guard flag), (c) if retry falls through to fallback engine, `_inFallback` is re-set, (d) the user can still manually retry via the existing Retry AI button.
