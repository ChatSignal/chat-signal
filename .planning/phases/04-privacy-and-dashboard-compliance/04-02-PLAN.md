<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure GitHub Pages and DNS for chatsignal.dev</name>
  <action>
The user must perform two manual steps that Claude cannot do via CLI:

**Step 1: Enable GitHub Pages in repo settings**
1. Go to https://github.com/johnzilla/chat-signal-radar/settings/pages
2. Under "Build and deployment" → Source: select "Deploy from a branch"
3. Branch: select `main`, folder: select `/docs`
4. Click Save
5. Under "Custom domain": enter `chatsignal.dev`, click Save
6. Check "Enforce HTTPS" (may need to wait for certificate provisioning)

**Step 2: Configure DNS at your domain registrar for chatsignal.dev**
Add four A records pointing to GitHub Pages IPs:
```
A  @  185.199.108.153
A  @  185.199.109.153
A  @  185.199.110.153
A  @  185.199.111.153
```

Note: DNS propagation can take up to 24-48 hours. HTTPS certificate provisioning may take an additional few hours after DNS resolves.
  </action>
  <how-to-verify>
After DNS propagates, verify with:
```bash
dig chatsignal.dev +noall +answer -t A
```
Should return all four GitHub Pages IPs.

Then open in a browser:
```
https://chatsignal.dev/privacy-policy
```
Should display the privacy policy rendered as HTML by Jekyll.
  </how-to-verify>
  <resume-signal>Type "live" when https://chatsignal.dev/privacy-policy loads correctly in a browser, or describe any issues.</resume-signal>
</task>
