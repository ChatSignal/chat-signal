Make two changes to `extension/sidebar/sidebar.js`:

**Change 1: Add `checkStorageAvailability()` function**

Add this function near the top of the file (after the `loadSettings` function or near `checkAISettings`). It checks if there is enough space for the WebLLM model download:

```javascript
const REQUIRED_BYTES = 450 * 1024 * 1024; // 450MB

async function checkStorageAvailability() {
  if (!navigator.storage || typeof navigator.storage.estimate !== 'function') {
    // API unavailable — allow attempt (per decision: let it fail naturally)
    return { sufficient: true };
  }
  try {
    const { quota, usage } = await navigator.storage.estimate();
    const available = quota - usage;
    return { sufficient: available >= REQUIRED_BYTES };
  } catch (err) {
    console.warn('[LLM Consent] Storage estimate failed:', err);
    return { sufficient: true }; // On error — allow attempt
  }
}
```

Key behaviors per locked decisions:
- 450MB threshold (model size + IndexedDB overhead buffer)
- If `navigator.storage.estimate()` API is unavailable → return sufficient: true (allow attempt)
- If the API call throws → return sufficient: true (allow attempt, let it fail naturally)

**Change 2: Modify `checkAISettings()` to await storage check before showing modal**

In the `checkAISettings()` function, modify the "First time - show consent modal" branch (the `else` branch around line 250-252). Instead of immediately showing the modal, first check storage availability, then apply the result before showing the modal.

Replace the else branch:
```javascript
// Current code:
} else {
  // First time - show consent modal
  statusText.textContent = 'Ready! Waiting for chat messages...';
  llmConsentModal.classList.remove('hidden');
}
```

With:
```javascript
} else {
  // First time — check storage then show consent modal
  statusText.textContent = 'Ready! Waiting for chat messages...';

  const { sufficient } = await checkStorageAvailability();
  if (!sufficient) {
    llmEnableBtn.disabled = true;
    const warningEl = document.getElementById('llm-space-warning');
    if (warningEl) warningEl.classList.remove('hidden');
    llmSkipBtn.textContent = 'Continue without AI';
  }

  llmConsentModal.classList.remove('hidden');
}
```

This awaits the storage check BEFORE removing 'hidden' from the modal, preventing the race condition where the user could click "Enable AI" before the check completes (per research pitfall #4). When space is insufficient:
- "Enable AI" button is disabled (greyed out via CSS :disabled)
- Warning message "Not enough disk space available (~450MB needed)" becomes visible
- "Skip for now" button text changes to "Continue without AI"

When space IS sufficient (or API is unavailable), the modal shows normally with both buttons enabled and default text.

Do NOT show the `navigator.storage.estimate()` result in the modal text — per locked decision, only state the ~450MB requirement statically.
